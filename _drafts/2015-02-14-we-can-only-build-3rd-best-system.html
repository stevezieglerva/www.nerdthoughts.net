--- layout: post title: We Can Only Build the 3rd Best System date: '2015-02-14T10:00:00.000-05:00' author: Stephen Ziegler tags: modified_time: '2015-02-14T10:00:04.265-05:00' blogger_id: tag:blogger.com,1999:blog-1727955271917225446.post-9155563601227685339 blogger_orig_url: http://www.nerdthoughts.net/2015/02/we-can-only-build-3rd-best-system.html --- In the early 2000s, I worked on a large project to replace a set of mainframe systems with my company's fancy new client-server application. It was a great learning experience implementing an enterprise system for a huge, global company. I worked with some great clients, especially the older COBOL programmers of the legacy systems.

They had a great quote about building systems that has always stuck with me:
"We can only build the 3rd best system. We can't afford the best system and we won't know the 2nd best system until we build it."
I love the sense that of hindsight that the quote implies. No matter how much planning and good execution you put into building something, you can always to it better the next time.
