--- layout: post title: I Saw Something Awesome Today date: '2013-10-18T23:09:00.000-04:00' author: Stephen Ziegler tags: - agile - continuous integration modified_time: '2013-10-18T23:11:36.014-04:00' thumbnail: http://4.bp.blogspot.com/-FpAmBEU5uU8/UmHwvLWVA0I/AAAAAAAAgwQ/smh_Dtfzrvo/s72-c/Brookland-20131018-00016.jpg blogger_id: tag:blogger.com,1999:blog-1727955271917225446.post-7686552761067591946 blogger_orig_url: http://www.nerdthoughts.net/2013/10/i-saw-something-awesome-today.html --- Today, I was at InnovateVA where ICF Interactive had a sponsor booth. Guy Winterbotham (@guywinterbotham) showed off an epic build monitor.


First, Guy described how his team for a large Federal client built a continuous integration (CI) process.  The process kicks off after each code check-in by a developer and automatically:
  • Attempts to build the software by compiling the code and linking dependencies
  • Automates the execution of an entire suite of unit tests
  • Automates code inspection to ensure the code meets the team's format, naming, and documentation conventions
  • Notifies developers and managers via IM if the build is broken
That is awesome.

Now, Guy is a big nerd (said by someone blogging about CI on a Friday night) and added both a homemade pirate ship desktop cover and a gamification front-end. The pirate ship includes a huge light pirate that turns different colors depending on the build status:
  • Green = Build works
  • Purple = Some unit tests fail
  • Red = Build won't even compile

I like how the PVC pipe gives it a steampunk feel.

Guy also added a gamified dashboard that shows current build score, last module to screw up the build, and a ranking of devs by quality.


That is awesomer.

So, to put this in context for my fellow colleagues outside of IT, it's like having a 15 person team (proposal manager, proposal coordinator, section authors, SMEs, and reviewers) working on a 100 page proposal.  Upon save, it would be like automatically:
  • Informing all other team members what exactly was changed and when
  • Checking all spelling and grammar
  • Ensuring that all text meets the team's conventions (name of the company team, "website" vs "web site", bullet styles and punctuation)
  • Re-checking each item in the compliance matrix
  • Checking that document meets all page limit requirements
  • IMing the entire team if someone screwed up the document
Wouldn't that be nice!
