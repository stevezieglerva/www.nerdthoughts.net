I work for a consulting firm where I need to charge my time to a set of charge codes.  On a given day, I could charge up to 10 different project numbers.  Some of the bigger projects have really complex charge codes to handle the complex cost accounting.

I realized that the desktop search that comes with Windows 7 could be used to create a simple full text lookup of the charge codes.  I could write a quick batch script to create a series of simple text files, named with a given pattern, that would be indexed by the OS.

I created a master project number text file that listed the project number and then the name of the project:

100001.1.1 - Vacation
100383.3.3.1 - ACME Website - Task 1 Requirements
100383.3.3.2 - ACME Website - Task 2 Design
100383.3.3.3 - ACME Website - Task 3 Development
101617.1 - Widgets R Us - Phase 1
101617.2 - Widgets R Us - Phase 2

I then created a simple AWK script to create a text file in a specific directory for each line.  The text files had the same name as the lines in the master project number text file.  I gave the file a unique file extension "pnl" (project number list) to help me narrow the search results.


BEGIN{
}
{
 print $0 > ".\\" newdir "\\" $0 ".pnl"
}
END{
}
